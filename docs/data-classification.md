# 数据分类与隐私合规（GDPR / PIPL / CCPA）

> 维度 5（供应链与合规）。本文档定义数据分类、PII 字段、保留期与删除机制，映射到代码实现。
> 企业为何需要：合规审计（SOC 2 / ISO 27001 / GDPR）要求"知道你持有哪些个人数据、存多久、如何删除"。

## 1. 数据分类

| 级别                    | 定义                  | 本系统实例                                                            |
| ----------------------- | --------------------- | --------------------------------------------------------------------- |
| **PII（个人身份信息）** | 可直接/间接识别自然人 | `users.username`、`users.last_login_at`、审计日志中的用户标识         |
| **敏感凭证**            | 泄露即危害安全        | `users.password_hash`（argon2id，不可逆）、JWT/Refresh Token（Redis） |
| **业务数据（非个人）**  | 不关联自然人          | 行情价格、回测结果、组合配置（除非含可识别命名）                      |
| **运营元数据**          | 系统遥测              | Prometheus 指标、pino 日志（已脱敏）、trace                           |

## 2. PII 字段清单（`migrations/004_users.sql`）

| 字段                 | 分类        | 处理方式                                             |
| -------------------- | ----------- | ---------------------------------------------------- |
| `username`           | PII         | 明文存储（登录标识）；删除时匿名化为 `deleted_<id8>` |
| `password_hash`      | 敏感凭证    | argon2id（mem 64MB/time 3）；删除时清空              |
| `last_login_at`      | PII（行为） | 时间戳；随账户删除一并处理                           |
| `role` / `is_active` | 非 PII      | 授权状态                                             |

> 系统**不收集** email、电话、地址、支付信息——遵循数据最小化原则（GDPR Art.5(1)(c)）。

## 3. 数据生命周期

| 数据               | 保留策略                                   | 删除机制                                                        |
| ------------------ | ------------------------------------------ | --------------------------------------------------------------- |
| 用户账户           | 活跃期间保留                               | `anonymizeUser`（默认，保留引用完整性）/ `deleteUser`（硬删除） |
| Refresh Token      | TTL `JWT_REFRESH_TTL`（默认 7 天）自动过期 | 登出即撤销；账户删除时撤销整个 token family                     |
| 审计日志（outbox） | 建议 ≥180 天（合规留存）                   | 关联用户匿名化后，日志保留但 userId 已不可关联到自然人          |
| 价格/回测缓存      | TTL 3600s                                  | 自动过期；非个人数据                                            |

## 4. 法规义务 → 代码映射

| 义务             | 法规                   | 实现                                               |
| ---------------- | ---------------------- | -------------------------------------------------- |
| 被遗忘权（删除） | GDPR Art.17 / PIPL §47 | `userService.anonymizeUser` / `deleteUser`         |
| 数据最小化       | GDPR Art.5             | 仅采集 username + hash，无多余 PII                 |
| 访问可追溯       | SOC 2 / GDPR Art.30    | 审计日志（HMAC 防篡改，ADR + auditLog.ts）         |
| 凭证保护         | GDPR Art.32            | argon2id、TLS（生产）、日志脱敏（logSanitizer.ts） |
| 假名化           | GDPR Art.4(5)          | 匿名化保留统计、抹除身份                           |
