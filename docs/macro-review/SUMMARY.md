## 3. 架构决策评估

### 应保持（keep）的决策

| 决策                        | ADR     | 评估                                                               |
| --------------------------- | ------- | ------------------------------------------------------------------ |
| Go 引擎 + Node 规范引擎双轨 | 008/031 | ✅ 合理。Node 规范引擎在 tactical/signal/pca/letf 等场景是权威实现 |
| Express + TypeScript        | 004     | ✅ 成熟度匹配项目需求                                              |
| pnpm workspace              | -       | ✅ 收益正向，3 包时虽不大但随扩展会放大                            |
| PostgreSQL + Redis          | 007/018 | ✅ 当前 ROI 负但先发选型避免了未来大迁移成本                       |
| Zod v4                      | 009     | ✅ 基础设施良好，核心计算路由全部覆盖                              |
| DDD 分层                    | 013     | ✅ Domain 层纯净，3 处违规为已知 pragmatism                        |
| Provider 接口抽象           | -       | ✅ 新增数据源仅 3-5 Go 文件，影响面控制良好                        |
| 多租户 RLS                  | 032     | ✅ 扩展性强，新增租户类型成本低                                    |
| Outbox 模式                 | 014/024 | ✅ 事务边界完整，at-least-once 保底                                |

### 应更改（change）的决策

| 决策                      | 问题                             | 建议                               |
| ------------------------- | -------------------------------- | ---------------------------------- |
| 优化器网格搜索位置        | Node 实现 vs Go 实现不一致       | 统一到 Go 引擎                     |
| 路由文件结构              | `backtestRoutes.ts` 5 合一       | 按策略类型拆分                     |
| 前端 store 使用模式       | 6 页面绕过 store 直接调 API      | 创建 admin store                   |
| `dataService.ts` 返回类型 | 纯数据类型不携带 degraded 元信息 | 改为 `{ data, degraded }` 包装类型 |

### 需深入调研（investigate further）的决策

| 决策               | 原因                             | 建议                                |
| ------------------ | -------------------------------- | ----------------------------------- |
| React 19 升级      | 无阻断但需验证 react-i18next v17 | Q3 做 feature branch 试升级         |
| Go 引擎跨服务契约  | OpenAPI 缺失 + 手动构造请求体    | 评估 protobuf/buf vs 仅补充 OpenAPI |
| WebSocket/实时支持 | 15-20 人天改造成本               | 有业务需求时做 POC                  |
| 前端子目录规划     | 22 页面 64% 在根目录             | 统一规划后一次性迁移                |
