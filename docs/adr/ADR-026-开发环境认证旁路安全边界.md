# ADR-026: 开发环境认证旁路安全边界

| 字段 | 值                          |
| ---- | --------------------------- |
| 编号 | ADR-026                     |
| 状态 | 已接受                      |
| 日期 | 2026-06-25                  |
| 范围 | `api/middleware/jwtAuth.ts` |

## Context

开发模式下，当 `JWT_SECRET` 为默认值时，`jwtAuth` 自动注入 `role: admin` 用户，
若 staging 误用默认密钥，等同内置后门（垂直审计 T-32 / HIGH）。

## Decision

1. 旁路仅当 `NODE_ENV=development` **且** `DEV_SKIP_AUTH=true` 显式开启。
2. 注入用户角色固定为 `readonly`，**永不** `admin`。
3. 生产环境忽略 `DEV_SKIP_AUTH`。

## Consequences

- 本地联调：在 `.env` 设置 `DEV_SKIP_AUTH=true` 可零摩擦跳过认证。
- 安全：消除默认 admin 后门；RBAC 测试需使用真实 JWT 或 x-api-key。
