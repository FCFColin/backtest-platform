# ADR-003: Rust 为主引擎 Node.js 为 Fallback

> **企业理由**：回测平台的用户价值核心是计算速度和精度。Rust 在数值计算上的性能优势（10-100 倍）直接决定用户体验，但 Rust 引擎的可用性不能成为单点故障。Node.js 备用引擎确保系统在 Rust 不可用时仍可提供服务，牺牲速度换取可用性。

## Context（背景和驱动力）

回测平台的计算引擎面临以下挑战：

1. **性能需求**：蒙特卡洛模拟 10 万次路径、组合优化（有效前沿计算）、大规模矩阵运算（nalgebra + rayon 并行），Rust 的零成本抽象和 SIMD 优化在这些场景下性能远超 Node.js。
2. **可用性需求**：Rust 引擎作为独立进程，可能因 OOM、编译错误、Docker 重启等原因不可用，不能让整个平台因此瘫痪。
3. **精度需求**：Rust 引擎使用 nalgebra 进行精确矩阵运算；Node.js 备用引擎在 drag（税收拖累）等高级功能上精度较低。
4. **已有实现**：Node.js 端已有完整的回测逻辑（`api/engine/` 目录），是 Rust 引擎的原型实现，功能对等但性能差。

降级策略的演进：
- **v0.1**：简单 try-catch，Rust 失败直接走 Node.js，无重试、无熔断。问题：偶发超时触发 30s 全量降级。
- **v0.2（当前）**：引入 opossum 熔断器（三态模型）+ 指数退避重试，详见 `api/utils/rustFallback.ts`。

## Decision（决策内容）

### 调用链路

```
前端请求 → Node API → callRustWithFallback()
                        ├─ 1. 指数退避重试（最多 2 次，base 200ms + jitter）
                        │   └─ 通过 opossum 熔断器调用 Rust 引擎
                        │       ├─ Closed 状态：正常放行
                        │       ├─ Open 状态：快速失败（~0ms），直接走 fallback
                        │       └─ HalfOpen 状态：放行少量请求探测恢复
                        └─ 2. Rust 不可用时降级到 Node.js 备用引擎
```

### 熔断器配置

| 参数                       | 值    | 说明                           |
| -------------------------- | ----- | ------------------------------ |
| timeout                    | 5000ms | 单次请求超时                   |
| errorThresholdPercentage   | 50%   | 错误率超过 50% 时熔断          |
| resetTimeout               | 30s   | 熔断后 30s 进入半开状态探测    |
| volumeThreshold            | 5     | 至少 5 次请求才开始计算错误率  |

### 降级提示

降级时向客户端返回 `degradedWarning` 字段，告知用户当前使用备用引擎及精度差异：
- `BASE`：降级模式，使用 Node.js 备用引擎
- `WITH_DRAG`：降级且含 drag 近似计算，精度可能略低
- `WITHOUT_DRAG`：降级，drag 等高级功能精度较低

### 引擎间鉴权

Node → Rust 通信使用启动时生成的 `ENGINE_SHARED_SECRET`（`crypto.randomBytes(32)`），通过 `X-Engine-Auth` 请求头传递，防止未授权调用 Rust 引擎。

## Consequences（后果）

### 正面

- **性能优先**：正常情况下 Rust 引擎处理计算，蒙特卡洛 10 万次模拟 < 1s（Node.js 需 10-100s）。
- **高可用**：Rust 引擎故障时自动降级，用户无感知（仅看到降级提示），不丢失服务。
- **防雪崩**：熔断器在 Rust 引擎持续故障时快速失败，避免大量请求等待超时拖垮 Node API。
- **自愈能力**：指数退避重试让瞬态故障（网络抖动、短暂 GC 停顿）自愈，避免不必要的降级。

### 负面

- **精度不一致**：Node.js 备用引擎在 drag 等高级功能上精度较低，用户可能在不同时间得到不同结果。
- **代码双份维护**：回测逻辑在 Rust 和 Node.js 中各有一份实现，功能对齐和 bug 修复需同步更新。
- **降级延迟**：熔断器 Open 状态下，所有请求直接走 Node.js，计算耗时增加 10-100 倍。
- **内存翻倍**：Node.js 备用引擎需要加载完整的计算逻辑到内存，增加约 50-100MB 内存占用。
