# ADR-035: 自助注册、邮箱验证与组织邀请（nodemailer）

> **企业理由**：SaaS 自助开通要求"邮箱即账号"——访客可自行注册并创建组织、邀请同事协作，无需人工开通。邮箱验证防滥用，邀请机制让团队按角色加入组织。

| 字段   | 值                                                                |
| ------ | ----------------------------------------------------------------- |
| 状态   | 已接受                                                            |
| 日期   | 2026-06-25                                                        |
| 决策者 | 架构组                                                            |
| 范围   | 身份、注册、协作                                                  |
| 关联   | ADR-017（认证授权）、ADR-032（多租户 RLS）、ADR-033（按组织密钥） |

## Decision（决策内容）

### 数据（迁移 `010_user_email.sql`）

- `users` 增 `email`（大小写不敏感唯一，迁移期可空）+ `email_verified_at`。
- `email_verification_tokens`（user_id、token_hash、expires_at、consumed_at）。
- `invitations`（org_id、email、role、token_hash、invited_by、expires_at、accepted_at）；同组织同邮箱仅允许一条待处理邀请。
- 三表属身份/控制平面，不启用 RLS（在"尚未解析租户"或"接受邀请跨租户"流程中被查询）。

### 邮件（`api/services/mailService.ts`，nodemailer）

- `EMAIL_TRANSPORT=console`（默认，开发把链接打到日志）| `smtp`（生产经 SMTP 投递）。
- 链接基于 `APP_BASE_URL` 拼接，指向前端验证 / 接受邀请页。

### 路由

- `POST /auth/register`：单事务创建用户 + 组织 + owner 成员，事务外发验证邮件（失败可重发）。
- `POST /auth/verify-email`、`POST /auth/resend-verification`。
- `/api/v1/orgs/*`：组织信息读/改（admin）、成员列表/改角色/移除（admin，拒绝降级或移除最后一个 owner）、邀请增删查（admin），以及 `POST /invitations/accept`（仅需登录，凭明文令牌建立 membership）。

### 前端

- `SignupPage`、`VerifyEmailPage`、`AcceptInvitePage`、`OrgMembersPage`，`authStore` 扩展 `register/acceptInvite`。

## Consequences（后果）

### 正面

- 完整自助开通与团队协作闭环；令牌仅存哈希、有过期、可吊销。
- console 传输让本地联调零外部依赖。

### 负面

- 邮件投递为外部依赖，生产需配置并监控 SMTP。
- 注册创建组织的事务跨多表，需保证原子性（已用单事务 + 唯一约束兜底）。
