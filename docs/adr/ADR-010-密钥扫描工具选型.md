# ADR-010: 密钥扫描工具选型（gitleaks vs detect-secrets vs truffleHog）

> **企业理由**：密钥一旦进入 git 历史便难以彻底清除，自动化密钥扫描是防止凭据泄露、满足合规审计要求的必备前置能力。

## Context
代码仓库可能意外提交API密钥、数据库凭据、JWT签名密钥等敏感信息。一旦推送到远程仓库，即使后续删除，git历史中仍可恢复，造成持续泄露风险。需要在pre-commit hook和CI流水线中集成密钥扫描能力，在敏感信息进入代码仓库前拦截。

候选方案：
- **gitleaks**：Go编写的密钥扫描工具，单二进制分发
- **detect-secrets**：Yelp开源的Python密钥扫描工具，基于正则+熵值检测
- **truffleHog**：Python编写的git历史密钥扫描工具，支持高熵字符串检测

## Decision
采用gitleaks作为密钥扫描工具。

## Consequences
- (+) 单二进制，无Python/Node依赖，pre-commit hook和CI均可零成本集成，不引入额外运行时环境
- (+) 规则库覆盖100+种密钥类型（AWS、GCP、Azure、Stripe、JWT等），且持续更新
- (+) 支持git历史扫描（`gitleaks detect`）和实时扫描（`gitleaks protect`），覆盖已有仓库和新增提交
- (+) GitHub Actions官方action可用（gitleaks/gitleaks-action），CI集成开箱即用
- (+) 支持自定义规则（.gitleaks.toml），可针对项目特有密钥格式扩展
- (-) 放弃detect-secrets——Yelp维护频率低（最近更新间隔长），正则+熵值组合策略误报率较高
- (-) 放弃truffleHog——配置复杂，v3重构后API不稳定，CI集成需要额外适配
- (-) gitleaks基于正则匹配，对自定义编码的密钥可能漏检，需配合自定义规则补充
