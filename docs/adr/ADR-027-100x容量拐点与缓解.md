# ADR-027: 100x 容量拐点与缓解

| 字段 | 值         |
| ---- | ---------- |
| 状态 | 已接受     |
| 日期 | 2026-06-25 |

## Context

个人项目默认单实例；企业需回答「流量 ×100 哪里先崩」。本项目为多服务（Node + Go + PG + Redis）。

## Decision

按瓶颈顺序规划：

1. Compute（Node 事件循环 + 引擎 CPU）→ 异步队列 + HPA + Go 引擎扩展
2. DB 连接池 → PgBouncer + 读写分离 + `pg_pool_waiting_count` 告警
3. 外部数据 API → 信号量 + 熔断 + 缓存
4. Redis → 高可用集群

详见 [`docs/architecture.md`](../architecture.md#100x-流量瓶颈分析adr-027)。

## Consequences

- 优势：容量评审与告警有依据
- 劣势：部分缓解（Redis HA）需运维投入
