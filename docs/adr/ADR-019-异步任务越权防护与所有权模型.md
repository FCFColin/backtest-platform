# ADR-019: 异步任务越权防护与所有权模型

> **企业理由**：异步任务结果属于提交者的私有数据，缺乏所有权绑定会导致水平越权（IDOR / OWASP API1 BOLA），任意调用方可遍历 jobId 读取他人回测组合、参数与失败原因，造成数据泄露与合规风险。

## Context

参数优化（`/api/v1/backtest-optimizer/optimize`）与战术网格搜索（`/api/v1/tactical-grid/search`）
通过 BullMQ 异步执行，客户端经 `GET /api/v1/jobs/:id` 轮询结果。

审计发现（`docs/audit-vertical-v2.md` 维度8 / A01，CRITICAL）：

- `GET /api/v1/jobs/:id` 挂载 `optionalJwtAuth`，允许匿名访问（`api/app.ts:316`）。
- 入队时未记录提交者身份（`backtestOptimizerRoutes.ts:346-349`）。
- 处理器返回原始 `failedReason`，可能泄露引擎/栈内部细节（`jobRoutes.ts:36`）。

后果：未认证攻击者只需遍历或猜测 jobId（BullMQ 自增/可枚举）即可读取任意用户的任务结果。

## Decision

引入"任务所有权（ownership）"模型：

1. **入队写入 owner**：`BacktestJobData.userId` 记录提交者 `req.user.sub`（已存在字段）。
2. **查询强制认证**：`/api/v1/jobs` 由 `optionalJwtAuth` 升级为 `jwtAuth`。
3. **所有权校验**：仅 `job.data.userId === req.user.sub` 或 `req.user.role === 'admin'` 可读取；
   否则返回 **404**（与"任务不存在"同构，避免通过状态码差异枚举有效 jobId）。
4. **输出边界收敛**：失败任务不回传 `failedReason`，仅返回通用 `Job execution failed`，真实原因记入服务端日志。

## Consequences

- 优势：消除 IDOR；任务结果遵循最小权限；失败细节不外泄。
- 劣势：匿名提交的任务无法经本端点回取——可接受，因生产环境 compute 端点强制认证（ADR：T-03）后不存在匿名任务。
- 测试：新增越权访问（他人 jobId → 404）与所有者访问（200）回归用例。
- 工程权衡：用 404 而非 403 表达"无权访问"，牺牲语义精确性换取防枚举（OWASP 推荐）。
