# ADR-040: 属性测试

> **企业理由**：现有测试全部为 example-based（给定输入→期望输出），这类测试只能覆盖显式编写的用例，无法保证不变量（如对称性、幂等性、单调性）在所有输入空间上成立。对于金融计算逻辑，边缘情况（极端权重、负价格、空序列）容易遗漏。

## Decision（决策内容）

### 引入 fast-check

将 `fast-check` 添加为 devDependency，对核心不变量编写属性测试。

### 测试目标（`tests/property/`）

1. **组合权重归一化**：任意正权重数组经 `normalizeWeights()` 后，和恒为 1，元素均在 [0, 1] 区间。
2. **波动率正定性**：任意价格序列经 `calculateVolatility()` 后，输出严格为正。
3. **协方差矩阵对称性**：任意收益率矩阵经 `estimateCovariance()` 后，矩阵对称且半正定。
4. **排序单调性**：`rankByMomentum()` 输出顺序与输入收益率单调递减一致。
5. **再平衡幂等性**：已满足目标权重的组合经 `rebalance()` 后权重不变。

### 生成器策略

使用 fast-check 的 `fc.array(fc.float())`、`fc.record()` 等组合子生成合法输入空间，通过 `fc.pre()` 过滤无效输入（如空数组、全零序列）。

### CI 集成

属性测试作为 CI optional 层的一部分（见 ADR-038），每次提交执行 ~500 runs。局部运行可通过 `npx vitest run tests/property/` 执行。

## Consequences（后果）

### 正面

- 不变量覆盖率由手工用例扩展到自动生成的随机输入空间
- 发现 example-based 测试遗漏的边缘情况（如单资产组合、极端杠杆）
- 与 invariant() 断言（ADR-039）配合形成双重保障：开发期断言捕获 + 测试期随机生成发现

### 负面

- CI 增加约 10s 执行时间
- 属性测试非确定性：种子固定的用例保证可重现，但非固定种子运行可能因随机性产生失败
- 需要额外维护生成器与输入约束（`fc.pre`），确保生成的输入在业务语义上合法
